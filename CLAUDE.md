# Project Overview
This repository is used for research and data analysis projects.

# Goals
- Analyze and process datasets
- Generate insights and summaries from data
- Produce clean, well-documented scripts and outputs

# Code Style
- Code in Stata
- Write clean, readable code with comments explaining the logic
- Use descriptive variable names
- Keep functions small and focused on a single task

# File Organization
- Raw data files should not be modified — always work on copies
- Store outputs (charts, reports, processed data) in an `outputs/` folder
- Keep scripts in a `scripts/` folder

# When Writing Code
- Always include error handling for file reading/writing
- Add comments explaining what each section of code does
- Prefer simple, explicit code over clever one-liners

# Testing
- Before finalizing any script, verify it runs without errors
- Check that outputs match expectations with a small sample of data first

# Notes
- When in doubt, ask for clarification rather than making assumptions
- Prioritize reproducibility — code should produce the same results every time
